How to Install Rust on Windows Using the GNU Toolchain
(No Visual Studio / MSVC Required)

This guide shows you how to install Rust from scratch on Windows using the x86_64-pc-windows-gnu toolchain. This avoids the large Visual Studio Build Tools download and the common "link.exe not found" error.

Perfect for console apps, web backends (Axum, Rocket, etc.), and even simple native Windows GUI apps using Win32 API bindings.

Step 1: Clean Up Any Old Installation (Optional but Recommended)

  1. Open Settings → Apps → Installed apps.
  2. Uninstall anything named:
    • Rust or Rustup
    • Visual Studio
    • Microsoft C++ Build Tools
  3. (Optional) Delete leftover folders:
    • C:\Users\YourUsername\.rustup
    • C:\Users\YourUsername\.cargo
    (Enable "View hidden items" in File Explorer to see them.)

Step 2: Download and Run the Rust Installer

  1. Go to the official Rust website: https://www.rust-lang.org/tools/install
  2. Download rustup-init.exe (64-bit version).
  3. Double-click rustup-init.exe to run it.

Step 3: Choose the GNU Toolchain During Installation

When the console window opens:

1) Proceed with installation (default)
2) Customize installation
3) Cancel installation

> Type 2 and press Enter

Then set these options:

  • Default host triple: x86_64-pc-windows-gnu
    (This is the key change – it uses MinGW instead of MSVC)
  • Default toolchain: stable (recommended)
  • Leave everything else as default.

Press Enter or type 1 to continue the installation.

Note: Rustup will now download the GNU/MinGW-based toolchain (~300-800 MB). No Visual Studio or link.exe needed!

Step 4: Verify the Installation

  1. Open a new Command Prompt or PowerShell.
  2. Run these commands: 
    rustc --version
cargo --version
rustup show
  3. You should see: 
    Default host triple: x86_64-pc-windows-gnu
stable-x86_64-pc-windows-gnu (default)
  4. Test with a new project: 
    cargo new hello_rust
cd hello_rust
cargo run
    It should print "Hello, world!" with no errors.

You're Done!

You now have a clean Rust setup that:

  • Never requires Visual Studio or MSVC
  • Works great for console, web, and simple GUI apps
  • Avoids the infamous "link.exe not found" error forever

If you ever need the MSVC toolchain later, you can add it with:

rustup toolchain install stable-x86_64-pc-windows-msvc
